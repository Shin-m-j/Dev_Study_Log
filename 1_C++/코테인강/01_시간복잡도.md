# 시간 복잡도

## 1.알고리즘이란?

"유한한 수의 규칙에 따라 구별 가능한 **기호**들을 조작하여 입력한 정수에서 출력되는 정수를 생성"

### 1-1.알고리즘의 7가지 필수 성질

- 정밀성 : 모든 단계를 정확하게 **정의**하며, **정확한 결과값**이 도출 되어야 한다.
- 유일성 : 각 단계가 끝난후 다음 **수행할 단계**가 무엇인지 설계 되어야 한다.
- 타당성 : 이론으로만 가능이 아닌, 실제 컴퓨터 **자원내에서 구현** 가능해야 한다.
- 유한성 : 알고리즘은 반드시 원하는 값 도출시 종료 해야한다. (**무한루프 방지**)
- 일반성 : 틀정 데이터 하나에만 작동이 아닌, 정의된 **범위의 값에서 동작** 해야한다.
- 입력 : 알고리즘을 수행하는 데 필요한 데이터가 외부에서 입력 가능 해야한다.
- 출력 : 알고리즘 수행 후 최소 하나 이상의 결과을 도출 해야한다.

### 2-2.절대 시간 측정

절대 시간 측정은 실행 했을 때 결과 값이 도출 되는 시간을 측정

- 장점 : 같은 성능의 컴퓨터일 경우 확실한 성능이 나온다.
- 단점 : 각 컴퓨터의 성능에 따라서 시간이 달라진다.

### 2-3.연산 횟수 측정

코드가 실행되는 횟수를 기준으로 특정

- 장점 : 환경 영향을 받지 않아서 객관적인 지표로 사용 가능하다.
- 단점 : 입력값에 따라서 성능이 결정되어 입력되는 값의 기준이 필요하다.

이 단점 때문에 최악의 경우를 기준으로 성능을 측정한다.

### 2-4.내용 정리

- 코딩테스트에서 알고리즘 성능은 연산 횟수로 측정한다.
- 입력 값에 따라 연산 횟수가 다르다면 가장 최악의 경우를 측정한다.
- 입력 값에 따른 연산 횟수를 측정해서 성능의 지표로 나타내는 것이 시간 복잡도라고 한다.
